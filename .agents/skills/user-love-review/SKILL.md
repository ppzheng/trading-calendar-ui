# User Love Review

## Trigger

Use this skill when asked to review user experience, trust, clarity, retention value, feature usefulness, dashboard comprehension, or UI quality.

## Read first

- `AGENTS.md`
- `product/PRODUCT.md`
- `product/USERS.md`
- `product/CURRENT_STATE.md`
- `product/PRODUCT_RULES.md`
- `product/workflows/ui-optimization.md`
- Relevant UI and data files for the reviewed area.

## Review criteria

- Does the user immediately know the current month and state?
- Does the user understand the trading day classifications?
- Is today's information prominent enough?
- Are risk days visually and behaviorally warning-worthy?
- Does the user know what to do next?
- Is month switching clear?
- Is mobile viewing easy?
- Does calendar export provide real value?
- Is there repeated or overloaded information?
- Are there trust risks?

## Forbidden

- Do not modify code by default.
- Do not invent user research.
- Do not mark assumptions as facts.
- Do not recommend behavior changes that violate product rules.

## Output

Use this format:

- User goal
- Current experience
- Strengths
- Friction
- Trust risks
- Aha moment
- Retention value
- Recommendations
- Priority
- Evidence
- Assumptions
