# Product Decisions

Use this directory for durable product decisions that should guide future work.

Record a decision when it changes or clarifies product behavior, product rules, user-facing scope, data governance, localization expectations, risk/disclaimer handling, or accepted tradeoffs.

Do not record temporary discussion, unapproved ideas, or implementation notes as official decisions.

## Decision record format

```md
# DECISION-ID: Title

Status: Proposed | Approved | Deprecated
Date: YYYY-MM-DD

## Context

What problem, ambiguity, or tradeoff required a decision?

## Decision

What was decided?

## Alternatives

What other options were considered?

## Consequences

What changes because of this decision? What risks or follow-up work remain?
```
